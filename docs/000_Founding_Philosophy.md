# Founding Philosophy

> 우리는 영어를 가르치는 앱을 만드는 것이 아니다. 우리는 AI를 통해 인간의 장기 기억을 최적화하는 Personal Learning OS를 만든다.

## 1. 프로젝트의 출발점

Project Mnemosyne은 새로운 영어 콘텐츠를 만들기 위해 시작된 프로젝트가 아니다. 이미 시장에는 충분히 많은 영어 학습 콘텐츠가 존재하며, Cambridge의 *English Phrasal Verbs in Use* 역시 그중 매우 잘 구조화된 학습 자료이다. 그러나 좋은 콘텐츠가 존재한다는 사실은 학습자가 장기적으로 기억한다는 사실을 보장하지 않는다. 대부분의 학습자는 책을 끝까지 읽거나 문제를 풀어도 몇 주 또는 몇 달 후에는 핵심 표현을 잊고, 실제 대화 상황에서는 배운 표현을 즉시 떠올리지 못한다.

이 프로젝트의 출발점은 명확하다. 문제는 콘텐츠 부족이 아니라 기억의 실패이다. 따라서 우리는 콘텐츠를 단순히 디지털화하는 것이 아니라, 학습한 지식이 장기 기억으로 전환되고 실제 상황에서 사용할 수 있도록 만드는 학습 시스템을 구축한다.

## 2. 우리가 해결하려는 문제

기존 영어 학습 서비스는 대체로 `콘텐츠 제공 → 문제 풀이 → 점수 확인 → 다음 콘텐츠`의 구조를 가진다. 이 구조는 학습량과 활동량을 늘리는 데는 효과적일 수 있으나, 장기 기억을 보장하지 못한다. 학습자가 당일 문제를 맞혔다는 사실은 30일 뒤에도 그 표현을 회상할 수 있다는 의미가 아니며, 객관식 문제를 맞혔다는 사실은 실제 대화에서 그 표현을 자연스럽게 사용할 수 있다는 의미도 아니다.

우리가 해결하려는 문제는 “어떻게 더 많이 공부하게 만들 것인가”가 아니다. 우리가 해결하려는 문제는 “어떻게 더 오래 기억하게 만들 것인가”이며, 더 정확히는 “어떻게 기억된 지식을 실제 사용 가능한 지식으로 전환할 것인가”이다.

## 3. 우리가 믿는 것

우리는 학습보다 기억이 중요하다고 믿는다. 새로운 내용을 많이 접하는 것보다 이미 배운 내용을 잊지 않는 것이 더 중요하며, 복습은 신규 학습보다 우선되어야 한다. 우리는 Recognition보다 Recall이 중요하다고 믿는다. 정답을 보고 알아보는 능력보다, 아무 단서 없이 스스로 떠올리는 능력이 장기 기억에 더 직접적으로 연결된다.

우리는 모든 학습이 개인화되어야 한다고 믿는다. 같은 콘텐츠를 같은 순서로 공부하더라도 사람마다 기억하는 항목과 잊는 항목은 다르다. 따라서 학습 경로는 책의 순서가 아니라 학습자의 기억 상태에 의해 결정되어야 한다. 우리는 AI의 역할이 정답을 알려주는 것이 아니라 기억을 관리하고 학습자의 회상을 촉진하는 것이라고 믿는다.

## 4. 제품의 정체성

이 프로젝트는 영어 앱, 퀴즈 앱, 플래시카드 앱, Cambridge 교재 앱이 아니다. 이 프로젝트는 AI 기반 Personal Learning Operating System이다. 영어 phrasal verb는 첫 번째 적용 도메인일 뿐이며, 제품의 본질은 학습자의 기억 상태를 모델링하고, 그 상태에 맞춰 최적의 학습 활동을 추천하며, 장기적으로 사용할 수 있는 지식을 늘리는 데 있다.

제품의 핵심은 콘텐츠가 아니라 학습 운영체제이다. 콘텐츠는 입력값이고, OCR 데이터는 원천 데이터이며, Learning Object는 지식 단위이다. 그러나 최종적으로 관리해야 하는 것은 학습자의 Memory Object이다. 같은 `look up`이라는 표현이라도 학습자에게 익숙한 정도, 자주 틀리는 패턴, 회상 속도, 실제 사용 가능성은 모두 다르다. Project Mnemosyne은 이 차이를 중심으로 학습을 설계한다.

## 5. 최종 목표

우리의 목표는 사용자가 Unit 70을 끝내는 것이 아니다. 우리의 목표는 사용자가 6개월 후에도 phrasal verb를 자연스럽게 회상하고 사용할 수 있도록 만드는 것이다. 따라서 학습 완료율은 보조 지표일 뿐이며, 최종 성공 기준은 장기 사용 가능 지식의 증가이다.

이 프로젝트의 North Star는 Long-term Usable Knowledge이다. 학습자가 어떤 표현을 봤는지, 몇 문제를 풀었는지, 몇 Unit을 끝냈는지가 아니라, 시간이 지난 뒤에도 그 표현을 실제 상황에서 사용할 수 있는지가 가장 중요하다.

## 6. 모든 설계의 기준

앞으로 추가되는 모든 기능은 다음 질문을 통과해야 한다. 이 기능은 장기 기억을 향상시키는가? 이 기능은 Retrieval Practice를 증가시키는가? 이 기능은 개인화된 학습 경험을 강화하는가? 이 기능은 학습자의 기억 상태를 더 정확히 모델링하는가? 이 기능은 사용자의 학습 부담을 불필요하게 늘리지 않는가? 이 질문에 명확히 답하지 못하는 기능은 구현 우선순위를 낮춘다.

## 7. 우리의 약속

우리는 기능이 많은 제품을 만들지 않는다. 우리는 사용자가 가장 적은 노력으로 가장 오래 기억하도록 돕는 제품을 만든다. 복잡함보다 단순함을 선택하고, 콘텐츠보다 기억을 선택하며, 기술보다 학습 효과를 선택한다. AI는 중심이 아니라 수단이며, 중심에는 항상 학습자의 기억이 있다.

## 8. 프로젝트 선언문

우리는 영어를 공부하는 앱을 만드는 것이 아니다. 우리는 AI를 이용하여 사람의 기억을 관리하고, 학습한 지식을 장기적으로 사용할 수 있는 지식으로 전환하는 Personal Learning Operating System을 만든다.
